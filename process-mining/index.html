이벤트 로그부터 Petri Net까지
Process Mining 매트릭스

이벤트 로그에서 실제 프로세스를 발견하고, 설계와 얼마나 맞는지 검사하고, 병목을 찾아 개선하는 흐름을 Petri Net 같은 형식모델과 함께 정리했습니다.

참고. van der Aalst의 Process Mining: Data Science in Action을 뼈대로 구성했습니다. 알고리즘별 수식 증명 페이지는 다음 단계에서 이어갑니다.
프로세스 발견 적합성 검사 프로세스 향상 형식모델 동시성 패턴 심화 프로세스 마이닝 도구 예측적 프로세스 모니터링 심화 이벤트 로그 품질

대분류 안에 중분류, 그 안에 구체 기법 카드가 있는 구조입니다. 아직 빈칸 증명 페이지는 없고 개념 정리 단계입니다.

프로세스 발견 (Process Discovery)

이벤트 로그만 보고 실제 프로세스 흐름을 역으로 찾아내는 층입니다.

발견 알고리즘

Alpha Algorithm

이벤트 로그의 선후관계로 Petri Net 복원

Heuristic Miner

빈도 기반으로 노이즈에 강한 모델 발견

Inductive Miner

재귀적 분할로 항상 유효한 프로세스 트리 보장

적합성 검사 (Conformance Checking)

실제 실행 기록이 설계된 프로세스와 얼마나 맞는지 재는 층입니다.

검사 기법

Token Replay

로그를 모델 위에서 재생시켜 이탈 지점 탐지

Alignment 기반 적합성

로그와 모델 실행경로의 최소편집거리

프로세스 향상 (Process Enhancement)

발견한 모델을 실제 개선과 예측에 쓰는 층입니다.

분석 · 예측

병목구간 분석

대기시간이 긴 구간 식별

예측 모니터링

진행중인 케이스의 남은 시간 · 결과 예측

형식모델 (Formal Models)

프로세스를 수학적으로 정확하게 표현하는 층입니다.

표현 형식

Petri Net

플레이스 · 트랜지션 · 토큰으로 동시성 표현

BPMN

실무에서 널리 쓰는 비즈니스 프로세스 표기법

프로세스 트리

재귀적 연산자로 표현한 블록구조 모델

동시성 패턴 심화

Petri Net으로 병렬 · 분기 흐름과 그 오류 상태를 표현하는 방법입니다.

동시성 · 검증

AND-split/AND-join

하나의 트랜지션에서 여러 흐름이 동시에 갈라지고 다시 합쳐지는 병렬 구조

XOR 분기

여러 흐름 중 하나만 배타적으로 선택되는 조건 분기

데드락 · 라이브락 검증

더 이상 진행 불가능한 상태와 무한 순환 상태를 형식적으로 검증

프로세스 마이닝 도구

이론을 실제 이벤트 로그에 적용할 때 쓰는 소프트웨어 생태계입니다.

대표 도구

PM4Py

파이썬 기반 프로세스 마이닝 라이브러리

ProM

대표적인 오픈소스 프로세스 마이닝 툴킷

Celonis

상용 프로세스 마이닝 플랫폼

예측적 프로세스 모니터링 심화

진행중인 케이스의 다음 행동과 결과를 미리 예측하는 층입니다.

예측 기법

Next Activity Prediction

진행중인 케이스의 다음 이벤트를 예측

Remaining Time Prediction

케이스가 종료될 때까지 남은 시간을 예측

이상 케이스 조기 경고

정상 패턴에서 벗어난 케이스를 진행 중에 탐지

이벤트 로그 품질

분석 결과의 신뢰도를 좌우하는 이벤트 로그 자체의 품질 문제입니다.

로그 구조 · 품질 이슈

필수 3요소

케이스ID, 활동명, 타임스탬프로 구성된 최소 이벤트 구조

로그 불완전성

누락된 이벤트로 인해 실제 흐름이 왜곡되는 문제

노이즈 필터링

드물게 발생하는 비정상 경로를 제거해 모델 품질 향상
