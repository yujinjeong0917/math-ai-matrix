이벤트 로그부터 Petri Net까지
Process Mining 매트릭스

이벤트 로그에서 실제 프로세스를 발견하고, 설계와 얼마나 맞는지 검사하고, 병목을 찾아 개선하는 흐름을 Petri Net 같은 형식모델과 함께 정리했습니다.

참고. van der Aalst의 Process Mining: Data Science in Action을 뼈대로 구성했습니다. 알고리즘별 수식 증명 페이지는 다음 단계에서 이어갑니다.
프로세스 발견 적합성 검사 프로세스 향상 형식모델

대분류 안에 중분류, 그 안에 구체 기법 카드가 있는 구조입니다. 아직 빈칸 증명 페이지는 없고 개념 정리 단계입니다.

프로세스 발견 (Process Discovery)

이벤트 로그만 보고 실제 프로세스 흐름을 역으로 찾아내는 층입니다.

발견 알고리즘

Alpha Algorithm

이벤트 로그의 선후관계로 Petri Net 복원

Heuristic Miner

빈도 기반으로 노이즈에 강한 모델 발견

Inductive Miner

재귀적 분할로 항상 유효한 프로세스 트리 보장

적합성 검사 (Conformance Checking)

실제 실행 기록이 설계된 프로세스와 얼마나 맞는지 재는 층입니다.

검사 기법

Token Replay

로그를 모델 위에서 재생시켜 이탈 지점 탐지

Alignment 기반 적합성

로그와 모델 실행경로의 최소편집거리

프로세스 향상 (Process Enhancement)

발견한 모델을 실제 개선과 예측에 쓰는 층입니다.

형식모델 (Formal Models)
