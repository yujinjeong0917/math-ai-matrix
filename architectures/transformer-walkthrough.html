← 모델 아키텍처 심화로 돌아가기

트랜스포머 인코더, 입력이 출력이 되기까지

토큰 시퀀스 하나가 인코더 블록 하나를 통과하는 동안 텐서의 모양과 값이 실제로 어떻게 바뀌는지 한 단계씩 그대로 따라갑니다. 배치 크기는 생략하고 시퀀스 길이 $n$, 모델 차원 $d$, 헤드 수 $h$로 표기합니다.

1입력 토큰 → 임베딩

$(n,) \rightarrow (n, d)$

토큰 ID는 그 자체로는 연산할 수 없는 정수입니다. 각 ID를 학습 가능한 $d$차원 벡터로 바꾸는 조회 테이블(임베딩 행렬)을 거칩니다.

$X = \mathrm{Embed}(\text{tokens}) \in \mathbb{R}^{n\times d}$

2포지션 임베딩 더하기

$(n, d) \rightarrow (n, d)$

셀프어텐션 자체는 순서를 모릅니다. 토큰의 위치 정보를 같은 차원의 벡터로 만들어 그대로 더해줍니다.

$X \leftarrow X + PE,\quad PE\in\mathbb{R}^{n\times d}$

3Q, K, V로 투영

$(n, d) \rightarrow (n, d)\times 3$

각 토큰이 다른 토큰에게 무엇을 물어보고(Query) 무엇으로 응답하는지(Key), 실제로 전달할 내용은 무엇인지(Value)를 서로 다른 선형변환으로 만듭니다.

$Q=XW_Q,\ K=XW_K,\ V=XW_V$

4헤드 분할

$(n, d) \rightarrow (h, n, d/h)$

한 번에 하나의 관점으로만 attention을 주는 대신 $d$차원을 $h$개의 작은 조각으로 나눠, 서로 다른 관점을 동시에 학습하게 합니다.

$d_k = d/h$

5스케일드 닷프로덕트 어텐션

$(n, d_k) \rightarrow (n, d_k)$ (헤드마다)

각 토큰이 다른 모든 토큰과 얼마나 관련 있는지 내적으로 점수를 매기고, $\sqrt{d_k}$로 나눠 값이 과하게 커지는 것을 막은 뒤 소프트맥스로 가중치를 만듭니다.

$\mathrm{Attn}(Q,K,V)=\mathrm{softmax}\!\left(\dfrac{QK^T}{\sqrt{d_k}}\right)V$

6헤드 합치기 · 출력 투영

$(h, n, d_k) \rightarrow (n, d)$

헤드별 결과를 다시 옆으로 이어붙이고, 마지막 선형변환으로 원래 차원 $d$로 되돌립니다.

$O = \mathrm{Concat}(\text{head}_1,\dots,\text{head}_h)W_O$

7잔차연결 · 레이어정규화

$(n, d) \rightarrow (n, d)$

어텐션이 원본 정보를 완전히 지워버리지 않도록 입력을 다시 더해주고, 각 토큰 벡터의 평균과 분산을 맞춰 학습을 안정시킵니다.

$X \leftarrow \mathrm{LayerNorm}(X + O)$

8피드포워드 네트워크

$(n, d) \rightarrow (n, d_{ff}) \rightarrow (n, d)$

토큰마다 독립적으로 작동하는 작은 2층 신경망입니다. 한 번 차원을 크게 넓혔다가(보통 4배) 다시 줄이면서 비선형 표현력을 더합니다.

$\mathrm{FFN}(x) = W_2\,\sigma(W_1 x + b_1) + b_2$

9잔차연결 · 레이어정규화

$(n, d) \rightarrow (n, d)$

FFN 다음에도 같은 방식으로 잔차연결과 정규화를 한 번 더 거칩니다. 이 지점이 인코더 블록 하나의 최종 출력입니다.

$X \leftarrow \mathrm{LayerNorm}(X + \mathrm{FFN}(X))$

10블록 반복

$(n, d) \rightarrow (n, d)$, $N$번

3번부터 9번까지가 인코더 블록 하나입니다. 같은 구조를 $N$번 쌓으면 매 층마다 더 넓은 문맥을 반영한 표현이 만들어집니다.

$X^{(l+1)} = \mathrm{EncoderBlock}(X^{(l)}),\quad l=0,\dots,N-1$
