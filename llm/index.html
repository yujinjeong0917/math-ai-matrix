추론최적화 · 서빙

같은 모델을 더 빠르고 저렴하게 서빙하는 층입니다. 스케일링과 KV 캐시가 여기 속합니다.

KV 캐시 · 메모리

KV Cache

이전 토큰의 Key/Value 재사용으로 재계산 생략

PagedAttention

KV 캐시를 페이지 단위로 관리해 메모리 단편화 감소

Multi/Grouped-Query Attention

Key/Value 헤드 수를 줄여 캐시 용량 절감

프롬프트 캐싱

캐싱 비용 구조

총비용 = 캐시미스토큰×기본단가 + 캐시쓰기토큰×쓰기단가(기본가의 약 1.25배) + 캐시히트토큰×읽기단가(기본가의 약 0.1배)

캐시 히트율

히트율 = 캐시에서 읽은 토큰 수 / 전체 입력 토큰 수, 반복되는 시스템 프롬프트·few-shot 예시가 많을수록 상승

프리픽스 캐싱(Prefix Caching)

입력 앞부분(시스템 프롬프트·대화 히스토리)이 요청마다 동일하면 그 구간 KV 캐시를 재사용, 뒤에 붙는 새 토큰만 재계산

캐시 TTL

캐시가 유지되는 시간(보통 5분~1시간 수준), TTL 경과 시 캐시미스로 처리되어 전체 재계산

스케일링 · 양자화

스케일링 법칙(Chinchilla)

모델 크기 · 데이터량 · 연산량의 최적 비율

양자화(INT8/GPTQ/AWQ)

가중치를 저비트로 근사해 메모리 절감

스펙큘레이티브 디코딩

작은 모델의 초안을 큰 모델이 검증

에이전트 · 평가

LLM을 도구를 쓰는 주체로 확장하고, 여러 에이전트를 조율하고, 결과를 재는 층입니다.

에이전트 설계 패턴

ReAct

추론(Reasoning)과 행동(Action)을 교대로 반복하며 다음 행동을 결정

Reflexion

실패 원인을 언어로 자기반성한 뒤 다음 시도에 반영해 재시도

Plan-and-Execute

전체 계획을 먼저 수립한 뒤 하위 작업을 순차적으로 실행

Tree of Thoughts

여러 추론 경로를 트리로 탐색하며 평가 후 최선의 경로를 선택

도구 사용 · 함수 호출

Function Calling

모델이 사전 정의된 함수의 이름과 인자를 구조화된 형식으로 생성

Tool Use 스키마 설계

함수의 입출력 타입과 설명을 명세해 모델의 호출 정확도를 높임

MCP(Model Context Protocol)

도구·데이터 소스를 표준화된 방식으로 모델에 연결하는 개방형 프로토콜

메모리 · 상태관리

단기 vs 장기 메모리

단기는 컨텍스트 윈도우 안의 최근 대화, 장기는 벡터DB에 저장된 과거 정보

메모리 압축(요약 기반)

누적된 대화를 주기적으로 요약해 토큰 예산을 절약하며 핵심만 유지

에이전트 상태 머신

에이전트의 진행 단계를 명시적 상태와 전이 규칙으로 관리

멀티에이전트 오케스트레이션

Supervisor 패턴

하나의 에이전트가 하위 에이전트들의 작업을 조율하고 결과를 취합

역할 분담형 협업

코더·리뷰어처럼 역할이 다른 에이전트가 순차 또는 병렬로 협업

에이전트 간 메시지 프로토콜

에이전트끼리 작업을 주고받는 메시지 형식과 라우팅 규칙

대표 프레임워크 · 라이브러리

LangChain

체인·에이전트 구성을 위한 범용 LLM 오케스트레이션 프레임워크

LangGraph

상태 그래프 기반으로 에이전트 흐름을 명시적으로 제어하는 프레임워크

LlamaIndex

AutoGen

CrewAI

역할 기반 멀티에이전트 협업을 정의하는 프레임워크

Semantic Kernel

Microsoft가 만든 엔터프라이즈 통합 지향 에이전트 SDK

OpenAI Agents SDK

OpenAI가 제공하는 에이전트/도구 오케스트레이션 SDK

에이전트 평가

태스크 성공률(Success Rate)

궤적 평가(Trajectory Evaluation)

최종 답만이 아니라 중간 행동 순서와 도구 선택 과정을 채점

도구 호출 정확도

올바른 도구를 올바른 인자로 호출했는지 측정

LLM-as-judge

환각(Hallucination) 측정

컨텍스트 엔지니어링

긴 컨텍스트 안에서 정보를 어디에, 어떤 순서로 배치할지 설계하는 층입니다.

긴 컨텍스트 배치

Lost-in-the-middle

컨텍스트 중간에 위치한 정보는 양 끝에 놓인 정보보다 회상률이 낮아지는 현상

정보 배치 전략

중요한 정보를 프롬프트의 시작 또는 끝 부분에 배치해 회상률을 높임

컨텍스트 순서 민감도

같은 정보라도 제시 순서에 따라 모델 응답 품질이 달라짐

컨텍스트 구성 기법

시스템/유저/어시스턴트 역할 분리

역할별로 메시지를 구분해 지시와 정보를 명확히 나눠 전달

동적 컨텍스트 조립

검색 결과·도구 출력·대화 이력을 매 요청마다 새로 재구성

컨텍스트 토큰 예산 관리

모델의 최대 토큰 한도 안에서 시스템·이력·검색결과에 토큰을 배분

구조화 출력

모델의 출력 형식을 자유 텍스트가 아닌 정해진 구조로 강제하는 층입니다.

출력 형식 강제

JSON mode

모델이 유효한 JSON만 생성하도록 디코딩 단계에서 제약

함수 스키마 강제

함수 호출의 인자를 사전에 정의한 JSON 스키마에 맞춰 생성

타입 검증 파싱

생성된 출력을 타입 스키마로 파싱하고 실패 시 재시도

문법 제약 디코딩

Grammar-constrained Decoding

컨텍스트-프리 문법(CFG)에 맞는 토큰만 샘플링을 허용

정규식/유한상태 제약

정규식을 유한 오토마타로 변환해 토큰 단위로 후보를 필터링

구조화 출력의 트레이드오프

제약이 강할수록 형식 오류는 줄지만 표현력과 디코딩 속도가 저하될 수 있음

멀티모달 LLM

텍스트 외에 이미지 등 다른 모달리티를 입력·출력으로 함께 다루는 층입니다.

비전-언어 모델

Vision-Language Model(VLM)

이미지 인코더와 언어모델을 연결해 이미지에 대해 텍스트로 응답

CLIP류 정렬 사전학습

이미지와 텍스트 임베딩을 대조학습으로 같은 벡터 공간에 정렬

Cross-attention 결합

이미지 특징을 언어모델의 어텐션에 주입해 두 모달리티를 융합

이미지 토큰화

패치 토큰화

이미지를 고정 크기 패치로 분할해 각 패치를 토큰처럼 처리(ViT 방식)

이미지 토큰 수

해상도가 커질수록 이미지 토큰 수가 늘어 컨텍스트 예산을 많이 소모

타일링(Tiling)

고해상도 이미지를 여러 타일로 나눠 각각 인코딩한 뒤 결합
